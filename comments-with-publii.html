    Comments with Publii

    Publii is the CMS I use for this very ocassional blog. To be honest, I’m just kicking the tyres, testing out whether I really want to use a static website hosted on Github. So far so good, but obviously there isn’t much information here. Since you’re reading, though: welcome to a very exclusive club, and do let me know what you’re up to by emailing roguish at gmail dot com. I’m especially interested in your experiences of static web sites and other CMS options such as WordPress, whether as a reader, a writer, or a developer.

    That brings me to the subject of this post: comments.

    Until now, Publii was pretty bare-bones, and like all static websites, didn’t have a comments option. This is because comments require some kind of dynamic interactions that a static website just isn’t up to. Usually this involves Javascript, which static sites can’t easily handle.

    But now there’s a new version of Publii which has plugins. At this early point the plugins that are available will provide a static site with analytics and/or comments.

    I’m not a fan of Facebook comments or Disqus comments because of the way they suck up all your data and spread it around. They are great examples of the old saying that if you’re not paying, you’re the product. But there’s a plugin for a lightweight comment system I’d never heard of before, called Cusdis. I might just give it a go. On the other hand, it seems to be still in early development. Furthermore, I’m still not convinced of the benefits of a static site if it then also relies on non-static add-onns. Perhaps you might as well go back to a more dynamic CMS. Clearly I’m confused and still working this one out!

    Read the Publii 0.39 announcement for all the details.
