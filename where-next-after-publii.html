    Where next after Publii?

    Publii is the static site generator I've been testing out.

    A static site generator enables you to publish a website or blog without needing a database in the back-end. A site is really just a collection of pages that can be written in markdown. There's no interactive functionality that depends on Javascript or similar, hence 'static'.

    It's been interesting to see just how easy it is to publish a blog on GitHub Pages, using Publii as the CMS.

    But now I've tried it I can't help thinking that Publii is a little too constraining for my tastes. 

    Of course, there's a fine line between simplifying things, which Publii does well, and removing options. 

    Let's face it, this site isn't exactly the most beautiful thing in existence, but it's hard to improve its looks.

    Things are a little too hard to tweak, but I want to get the layout exactly as I want it, not as the CMS forces me.

    That means it's probably time to try a different static site generator.

    I'm thinking of trying out Hugo, which I can probably just plug into GitHub pages, although I seem to remember GitHub was promoting the use of Jekyll. 

    So this site might go quiet for a while, for me to concentrate on learning how to deploy a site with Hugo.

    And I'll report back as soon as there's something to report.
